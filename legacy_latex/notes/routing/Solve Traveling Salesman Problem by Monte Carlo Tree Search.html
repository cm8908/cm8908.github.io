Tree Search

트리 탐색에서 SSS﻿는 하나의 상태(노드)이다. 가령 루트 노드는 S={vstart}S=\{v_{start}\}S={vstart​}﻿, 단말 노드는 S=VS=VS=V﻿ 우리 목표는 후보 세트인 Sˉ\bar{S}Sˉ﻿에서 최선의 노드를 하나씩 선택해나가는 것. DFS, BFS 모두 높은 복잡도로 실용적이지 못하다. → MCTS의 사용

Monte Carlo Tree Search

주로 게임플레이 등의 결정과정에서 사용되는 휴리스틱 탐색 알고리즘. playout이라 불리는 프로세스를 여러 차례 반복하며, 각 playout은 다음 네 가지 스텝으로 구성됨

  • Selection: 루트 노드 R에서 시작해 R의 자식 노드를 하나 선택한다. 선택의 기준은 default policy를 따름. 새로 선택된 자식 노드를 루트 노드로 삼고 리프 노드 L에 도달할 때까지 반복
  • Expansion: L의 자식 노드를 한 개 이상 만들고 그 중 하나인 N을 선택함 (unless the game ends).
  • Simulation: N에서 시작해 random strategy를 통해 게임 종료까지 진행
  • Backpropagation: N~R에 이르는 경로상의 노드들의 정보를 업데이트함

Self-Learning

조합최적화 문제는 (1) labeled data의 품질에 영향을 받는데, (2) 고품질의 labeled 데이터를 얻는 것이 쉽지 않다는 문제가 있다. 따라서 본 논문에서는 강화학습 프레임웍을 사용해 위 수식의 hhh﻿ (value function)을 학습시킴

Reinforcement learning formulation

  • States: 방문한 노드의 ordered sequence SSS﻿. 위 신경망을 통해 lll﻿-차원 벡터의 그래프 임베딩으로 변환한다. 종료 상태는 S^\hat{S}S^﻿로 표기하며 모든 노드를 다 방문한 상태
  • Transition: 노드 v∈Sˉv\in \bar{S}v∈Sˉ﻿를 SSS﻿에 추가하는 것으로, deterministic transition이다
  • Actions: 노드 vvv﻿의 선택
  • Rewards: 상태 sss﻿에서 노드 vvv﻿를 선택(행동)했을 때 보상 함수 r(s,v)r(s,v)r(s,v)﻿는 다음과 같이 정의한다. 모든 노드가 방문되었을 때 tour length를 fff﻿, vvv﻿가 SSS﻿에 추가되었을 때의 partial tour length는 ggg﻿로 정의한다. 즉, 매 시점마다 full length와 partial length의 차이를 의미한다.
  • Policy: 다음 행동 vvv﻿를 선택하기 위한 정책으로는 MCTS를 사용한다.

에피소드 진행 중 매 시점의 (s,a,r) 튜플을 저장해 두었다가, 에피소드가 끝난 뒤 다음 MSE loss를 통해 신경망 업데이트가 진행된다. 신경망 hhh﻿는 reward rrr﻿을 근사하도록 학습된다.

결론

TSP에서 greedy하게 최단 거리 노드를 선택하는 것만 해도 팩토리얼 타임의 복잡도를 가지지만, 이 논문에서는 MCTS를 통해 신경망을 통해 “근사된” 최단 거리 노드를 선택하도록 하여 복잡도를 줄였다.
