SolidGen: An Autoregressive Model for Direct B-rep Synthesis

CAD 명령어 시퀀스는 sketch and extrusion에 제한되어있고, fillets나chamfer같은 명령 (B-rep기반이므로) 구현하지 못함

Index B-rep: ML 모델의 입력에 알맞게 numeric array로 나타낸 B-rep으로, 최소한의 정보만을 담고 있다. {V,E,F}\{V,E,F\}{V,E,F}﻿로 구성됨

Architecture

Indexed B-rep (B\mathcal{B}B﻿)에 대한 아래 확률 분포를 추정하는 것이 목표

이 때 zzz﻿는 class label과 같은 부가적인 정보에 대한 representation이다

Vertex model

  • Lexicographic order로 정렬됨 (z좌표, y좌표, x좌표 순서)
  • 1D list로 flatten

\def\V\mathcal{V}﻿Vseq={v0,v1,…,<EOS>}\mathcal{V}^{seq}=\{v_0,v_1,\dots,\text{<EOS>}\}Vseq={v0​,v1​,…,<EOS>}﻿, ∣Vseq∣=(∣V∣×3)+1|\mathcal{V}^{seq}|=(|\mathcal{V}|\times 3)+1∣Vseq∣=(∣V∣×3)+1﻿

  • 각 vtv_tvt​﻿값은 실수값 대신 6비트로 양자화된 값 사용
  • t={0,…,∣Vseq∣−1}t=\{0,\dots,|\mathcal{V}^{seq}|-1\}t={0,…,∣Vseq∣−1}﻿까지 autoregressive하게 다음 토큰을 예측함
  • 아래 임베딩을 거쳐 Transformer decoder에 입력됨

Transformer decoder는 next token vtv_tvt​﻿에 대한 확률 (logit)을 출력

Edge model

  • 각 edge는 Vertex indices로 구성됨
  • 각 edge E∈EE\in\mathcal{E}E∈E﻿를 먼저 정렬 후, E\mathcal{E}E﻿ 전체를 각 edge의 lowest vertex index에 따라 정렬함
  • 1D list로 flatten

Eseq={e0,e1,<NEW_EDGE>,…,<EOS>}\mathcal{E}^{seq}=\{e_0,e_1,\texttt{<NEW\_EDGE>},\dots,\texttt{<EOS>}\}Eseq={e0​,e1​,<NEW_EDGE>,…,<EOS>}﻿, ∣Eseq∣=∑E∈E∣E∣+1|\mathcal{E}^{seq}|=\sum_{E\in\mathcal{E}}|E|+1∣Eseq∣=∑E∈E​∣E∣+1﻿

  • Vertex values (x,y,z)에 대해 value embedding한 뒤, 특별 토큰인 EOS와 NEW_EDGE에 대해서도 임베딩하여 Concat한 뒤 Transformer encoder를 지남 ⇒ input embedding hinp∈\mathbf{h}_{inp}\inhinp​∈﻿ R(demb×∣(V∣+2))\mathbb{R}^{(d_{emb}\times |(\mathcal{V}|+2))}R(demb​×∣(V∣+2))﻿
  • 위에서 계산한 input embedding을 vertex index ej∈Eseqe_j\in\mathcal{E}^{seq}ej​∈Eseq﻿에 따라 gather한 뒤, positional embedding과 더해줌
  • 이후 Transformer decoder를 거쳐 pointer pt\mathbf{p}_tpt​﻿를 얻은 뒤, hinp\mathbf{h}_{inp}hinp​﻿에 레퍼런스하여 vertex indices에 대한 확률 분포를 얻음

Face model

Face는 edge에 대한 embedding으로 표현하며, edge는 vertex에 대한 embedding으로 표현하는 구조

  • Edge와 마찬가지로 1D list Fseq\mathcal{F}^{seq}Fseq﻿로 flattened, 각각은 edge indices로 구성됨
  • Edge model과 비슷하게 vertex embedding계산 (vertex index에 따라 gather), 특별 토큰인 EOS, NEW_FACE에 대한 embedding과 concat된 후 Transformer encoder를 지남 ⇒edge embedding hE∈\mathbf{h}_\mathcal{E}\inhE​∈﻿ R(∣E∣+2)×demb\mathbb{R}^{(|\mathcal{E}|+2)\times d_{emb}}R(∣E∣+2)×demb​﻿
  • 위에서 계산한 embedding을 다시 edge index에 따라 gather한 뒤 또 positional embedding과 더하여 또 다른 Transformer encoder를 지남
  • 이후 Transformer decoder를 거쳐 pointer를 얻은 뒤, edge embedding에 레퍼런스 하여 edge indices에 대한 확률 분포를 얻음
